## 1. Default Passwords

**Default credentials are one of the biggest security risks.**

* Manufacturers ship devices with well-known credentials

  * Examples: `admin/admin`, `admin/password`, `admin` with no password
* These credentials are:

  * Publicly documented
  * Known by attackers

### Best Practices

* **Change default passwords immediately**
* Use **long, strong, unique, complex passwords**
* Enable **Multi-Factor Authentication (MFA)** if supported
* Rotate administrative passwords about **every 90 days**
* Use a **password manager** to generate and store credentials

**Exam takeaway:**
Default passwords must always be changed because they are **public knowledge**.

---

## 2. Disable Unneeded Ports and Protocols

**Every open port is a potential entry point for attackers.**

* Devices enable many ports and protocols by default
* Many are **not required** for your organization

### Best Practices

* Audit all enabled ports and protocols
* Disable anything not required for operations
* Reduces the **attack surface**

### Use Secure Versions of Protocols

Replace insecure protocols with encrypted alternatives:

| Insecure    | Secure            |
| ----------- | ----------------- |
| HTTP (80)   | HTTPS (443)       |
| SMTP (25)   | SMTPS (465 / 587) |
| Telnet (23) | SSH (22)          |

**Exam takeaway:**
Disable unused ports and prefer **encrypted protocols** whenever possible.
