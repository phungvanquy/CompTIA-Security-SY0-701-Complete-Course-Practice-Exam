## How SSO Works (EXAM FLOW)

SSO relies on a **trusted relationship** between:

* **Identity Provider (IdP)**
* **Service Provider (SP)** (application or website)

### Simplified SSO Process

1. User logs into the **Identity Provider (IdP)**

   * Example: Active Directory, Google, Microsoft
2. User requests access to an SSO-enabled application
3. Application asks the IdP to verify the user
4. IdP sends an **authentication assertion**
5. Application grants access **without another login**
