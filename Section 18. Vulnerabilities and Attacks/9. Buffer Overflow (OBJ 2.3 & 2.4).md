## What Is a Buffer?

## Why Buffer Overflows Are Dangerous

* Overwritten memory may include **control data**
* Attackers can:

  * Redirect program execution
  * Run **malicious code**
  * Gain **remote code execution (RCE)**

ğŸ“Œ Buffer overflows are a **common initial attack vector**

## How a Buffer Overflow Attack Works (Conceptually)

1. Program allocates a fixed-size buffer
2. Input is **larger than expected**
3. Extra data spills into adjacent memory
4. Critical values (like return addresses) get overwritten
5. Program executes **attacker-controlled code**
